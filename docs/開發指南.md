# 🚀 Jekyll 部落格開發指南

## 📦 前置需求

### Windows 使用者
- 安裝 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
- 確保 Docker Desktop 正在運行

### Mac 使用者
- 安裝 [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/)
- 確保 Docker Desktop 正在運行

---

## 🎯 快速啟動

### Windows
雙擊執行 `start-dev.bat` 或在終端機執行：
```cmd
start-dev.bat
```

### Mac / Linux
在終端機執行：
```bash
chmod +x start-dev.sh  # 首次執行需要賦予執行權限
./start-dev.sh
```

---

## 🌐 訪問網站

啟動成功後，在瀏覽器開啟：
- **http://localhost:4000**
- **http://127.0.0.1:4000**

---

## ✨ 功能特色

### 🔥 熱重載（Hot Reload）
- 修改任何檔案（Markdown、HTML、CSS、JS）後，網頁會自動重新整理
- 不需要手動重啟伺服器

### ⚡ 增量編譯
- 使用 `--incremental` 參數，只編譯修改過的檔案
- 大幅提升編譯速度

### 📝 支援的檔案類型
- Markdown 文章（`_posts/*.md`）
- 草稿（`_drafts/*.md`）
- 佈局模板（`_layouts/*.html`）
- 樣式表（`_sass/*.scss`, `css/*.css`）
- JavaScript（`js/*.js`）
- 配置檔（`_config.yml`）

---

## 🛑 停止伺服器

按 `Ctrl + C` 即可停止伺服器

---

## 📝 撰寫新文章

### 1. 建立新文章
在 `_posts` 目錄下建立新檔案，檔名格式：
```
YYYY-MM-DD-文章標題.md
```

範例：
```
2026-01-04-我的新文章.md
```

### 2. 文章格式
```markdown
---
layout: post
title: "文章標題"
date: 2026-01-04 12:00:00 +0800
categories: [分類1, 分類2]
tags: [標籤1, 標籤2]
---

這裡是文章內容...

<!-- more -->

這裡是摘要後的內容...
```

### 3. 草稿功能
將文章放在 `_drafts` 目錄，不會被發布，方便您慢慢編輯

## 🔧 常見問題

### Q1: Docker 容器啟動失敗？
**解決方法**：
1. 確認 Docker Desktop 正在運行
2. 檢查 4000 端口是否被佔用
3. 重新啟動 Docker Desktop

### Q2: 修改檔案後沒有自動更新？
**解決方法**：
1. 檢查終端機是否有錯誤訊息
2. 手動重新整理瀏覽器（Ctrl+F5 強制重新整理）
3. 重新啟動開發伺服器

### Q3: 修改 `_config.yml` 後沒有生效？
**原因**：配置檔修改需要重啟伺服器
**解決方法**：按 `Ctrl+C` 停止，然後重新執行啟動腳本

### Q4: 中文顯示亂碼？
**解決方法**：確保所有 Markdown 檔案使用 UTF-8 編碼儲存

## 💡 小技巧

### 使用 VS Code 編輯
1. 安裝 VS Code
2. 安裝擴充套件：
   - Markdown All in One
   - Markdown Preview Enhanced
   - Code Spell Checker
3. 開啟專案資料夾，即可享受語法高亮和即時預覽

### 檢查文章格式
```bash

docker run --rm -v "%cd%":/work tmknom/markdownlint .
```
